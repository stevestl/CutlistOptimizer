Firebase Setup
  1. Create a Firebase project at console.firebase.google.com.
  2. Add a Web App and copy apiKey, authDomain, projectId, appId into DEFAULT_FIREBASE_CONFIG at the top of app.js.
  3. Enable Firestore Database (Native mode).
  4. Enable Authentication → Email/Password sign-in method.
  5. Add your domain (and localhost for testing) to Authentication → Authorized Domains.
  6. Paste the Firestore Security Rules below into your project's Firestore → Rules editor.
  7. Use the avatar menu to create your first user account, then promote it to Admin via Firestore Console (see Promoting a User to Admin below).

App Check (reCAPTCHA v3)

  1. Firebase Console → App Check → register your web app → choose reCAPTCHA v3.
  2. Google reCAPTCHA Admin (g.co/recaptcha/admin) → create a v3 site key for your domains (e.g. yourusername.github.io and localhost).
  3. Paste the site key into RECAPTCHA_SITE_KEY at the top of app.js.
  4. Start in Monitor mode; switch to Enforce after confirming legitimate traffic.

Recommended Firestore Security Rules

These rules allow admins to read all projects (needed for the Users & Projects panel above).

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {

    function isSignedIn() { return request.auth != null; }
    function isOwner(uid) { return isSignedIn() && request.auth.uid == uid; }
    function userDoc() {
      return get(/databases/$(database)/documents/users/$(request.auth.uid)).data;
    }
    function isAdmin() { return isSignedIn() && userDoc().role == 'admin'; }

    match /users/{uid} {
      allow read:   if isOwner(uid) || isAdmin();
      allow create: if isOwner(uid)
                    && request.resource.data.role == 'standard'
                    && request.resource.data.email == request.auth.token.email;
      allow update: if isAdmin();
      allow delete: if isAdmin();
    }

    match /projects/{projectId} {
      allow read:           if isAdmin()
                            || (isSignedIn() && request.auth.uid == resource.data.ownerUid);
      allow update, delete: if isSignedIn() && request.auth.uid == resource.data.ownerUid;
      allow create:         if isSignedIn() && request.auth.uid == request.resource.data.ownerUid;
    }
  }
}

Promoting a User to Admin

  1. Firebase Console → your project → Firestore Database.
  2. Browse to the users collection and find the user's document.
  3. Click the role field → change "standard" to "admin" → Update.
  4. The change takes effect on the user's next sign-in or page reload.

Security Notes

  • Firebase API keys are safe to embed in client-side code — they identify your project but do not grant access. Security is enforced by Firestore Rules and Authentication.
  • The Firestore rules prevent any client from self-assigning the admin role. Only the Firebase Console (service-account level) can promote users.

Hosting on GitHub Pages

  1. Push the repository to GitHub and go to Settings → Pages.
  2. Under Source, select Deploy from a branch → main / / (root) → Save.
  3. Add <your-username>.github.io to Firebase → Authentication → Authorized Domains.

Instructions

Quick Start

  1. Go to the Planning tab.
  2. In Model + Material Settings, select your OBJ file. Units are detected automatically from OBJ comments where possible; adjust manually if needed.
  3. Click Analyze Model to extract parts from the file.
  4. Review the Parts List. Adjust per-part thickness overrides or grain lock if needed.
  5. Click Plan Stock (Planning Result section) to optimize board purchases from the catalog range.
  6. Switch to Lumber Yard to re-optimize against specific boards at your yard.

Units + Orientation

  • Import units: mm, cm, m, in, or ft — detected automatically from OBJ file comments (Fusion 360 exports typically include unit info), or set manually.
  • Part outputs: metric (mm).
  • Stock inputs: imperial — thickness in quarters, width in inches, length in feet.

Part axes are normalized so:

  • X = longest dimension (grain runs along X).
  • Y = middle dimension.
  • Z = shortest dimension (thickness).

Raw · Net · Rough

  • Raw: bounding-box dimensions straight from the OBJ, before any allowances.
  • Net: finished target dimensions after axis normalization.
  • Rough: dimensions used for nesting optimization:
    • rough length = net length + length allowance
    • rough width = net width + width allowance
    • rough thickness = net thickness + thickness allowance

Thickness Assignment

Stock thickness is expressed in quarters (4/4 ≈ 25.4 mm, 5/4 ≈ 31.75 mm, etc.). If a part's rough thickness exceeds one board, multiple lamination layers are calculated automatically. You can override per-part or globally via the Override column and Apply Override button.

Planning vs Lumber Yard

  • Plan Stock (Planning tab): optimizes against any board width/length within the catalog range you configure (e.g., 4–12″ wide, 6–10 ft long). Good for budgeting.
  • Lumber Yard Recalculate: re-optimizes against the specific boards in your inventory table. Enable Assume infinite inventory to treat quantities as unlimited. With no inventory rows and infinite mode enabled, falls back to the planning catalog range.

Grain Direction

The Grain column in the Parts List controls how a blank is oriented during nesting:

  • Long — grain runs along the longest axis; the blank will not be rotated (default).
  • Mid — grain runs along the middle axis; the blank is pre-rotated 90° before nesting and rotation is then locked.
  • Free — nesting may rotate the blank freely to improve yield.

3D Preview

  • Drag to orbit · scroll/pinch to zoom · two-finger drag to pan.
  • Reset View returns to the auto-framed position.
  • Preview refreshes automatically after Analyze Model.

Project Save + Load

Sign in with an account (avatar button, upper-right) to save and load projects. Projects are stored in Firebase and accessible from any device using the same credentials.

Note: Firestore documents have a 1 MB limit. Very large OBJ files may fail to save; reduce mesh complexity in your CAD export if you hit this limit.

User Accounts

Click the avatar button (upper-right corner) to sign in, create an account, reset your password, or sign out. All accounts created through the web app are Standard users. The Admin tab is only visible to Admin users — contact your administrator to be promoted.

Additional Scale Multiplier

Applied after unit conversion: final_mm = value × unit_factor × multiplier. Use 1 for normal exports. Change only if your export is consistently scaled incorrectly.
