Instructions

Quick Start

  1. Go to the Planning tab.
  2. In Model + Material Settings, select your OBJ file. Units are detected automatically from OBJ comments where possible; adjust manually if needed.
  3. Click Analyze Model to extract parts from the file.
  4. Review the Parts List. Adjust per-part thickness overrides or grain lock if needed.
  5. Click Plan Stock (Planning Result section) to optimize board purchases from the catalog range.
  6. Switch to Lumber Yard to re-optimize against specific boards at your yard.

Units + Orientation

  • Import units: mm, cm, m, in, or ft — detected automatically from OBJ file comments (Fusion 360 exports typically include unit info), or set manually.
  • Part outputs: metric (mm).
  • Stock inputs: imperial — thickness in quarters, width in inches, length in feet.

Part axes are normalized so:

  • X = longest dimension (grain runs along X).
  • Y = middle dimension.
  • Z = shortest dimension (thickness).

Raw · Net · Rough

  • Raw: bounding-box dimensions straight from the OBJ, before any allowances.
  • Net: finished target dimensions after axis normalization.
  • Rough: dimensions used for nesting optimization:
    • rough length = net length + length allowance
    • rough width = net width + width allowance
    • rough thickness = net thickness + thickness allowance

Thickness Assignment

Stock thickness is expressed in quarters (4/4 ≈ 25.4 mm, 5/4 ≈ 31.75 mm, etc.). If a part's rough thickness exceeds one board, multiple lamination layers are calculated automatically. You can override per-part or globally via the Override column and Apply Override button.

Planning vs Lumber Yard

  • Plan Stock (Planning tab): optimizes against any board width/length within the catalog range you configure (e.g., 4–12″ wide, 6–10 ft long). Good for budgeting.
  • Lumber Yard Recalculate: re-optimizes against the specific boards in your inventory table. Enable Assume infinite inventory to treat quantities as unlimited. With no inventory rows and infinite mode enabled, falls back to the planning catalog range.

Grain Lock

When grain lock is on, a blank will not be rotated 90° during nesting — this preserves grain direction along the part's length. Toggle per-part in the Parts List, or set the default in Model + Material Settings.

3D Preview

  • Drag to orbit · scroll/pinch to zoom · two-finger drag to pan.
  • Reset View returns to the auto-framed position.
  • Preview refreshes automatically after Analyze Model.

Project Save + Load

Projects are saved to local browser storage by default (no server required). Connect Firebase in the Settings tab to sync to the cloud — projects are then accessible from any device using the same Firebase credentials.

Note: Firestore documents have a 1 MB limit. Very large OBJ files may fail to save to Firebase; reduce mesh complexity in your CAD export if you hit this limit.

Firebase Setup

  1. Create a Firebase project at console.firebase.google.com.
  2. Add a Web App and copy: apiKey, authDomain, projectId, appId.
  3. Enable Firestore Database (Native mode).
  4. Enable Authentication → Anonymous sign-in.
  5. Add your domain (and localhost for testing) to Authentication → Authorized Domains.
  6. Paste the config values into the Settings tab and click Connect Firebase.

Recommended Firestore Security Rules

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /projects/{projectId} {
      allow read: if request.auth != null
        && request.auth.uid == resource.data.ownerUid;
      allow update, delete: if request.auth != null
        && request.auth.uid == resource.data.ownerUid;
      allow create: if request.auth != null
        && request.auth.uid == request.resource.data.ownerUid;
    }
  }
}

Security Notes

  • Firebase API keys are safe to embed in client-side code — they identify your project but do not grant access. Security is enforced by Firestore Rules and Authentication.
  • Projects in localStorage are accessible to any JavaScript running on the same origin. Avoid storing sensitive data in project or part names.
  • Anonymous auth ties each project to an anonymous user ID. Switching browsers creates a new user and won't see prior projects unless you migrate ownerUid values.

Additional Scale Multiplier

Applied after unit conversion: final_mm = value × unit_factor × multiplier. Use 1 for normal exports. Change only if your export is consistently scaled incorrectly.
